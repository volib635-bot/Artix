# Artix — Vibe Coding & Prompt Engineering Playbook

> **Document Status**: Production-Ready  
> **Target Release**: v1.4.0  

---

## 🎯 1. Overview & Principles

Vibe Coding in Artix turns high-level feature ideas into surgical, file-precise AI prompts. To prevent AI code generators (Cursor, Artix, LLMs) from producing broken code, every prompt generated by Artix enforces **4 Core Rules**:

1. **Relative File Paths Only**: Always specify absolute/relative file targets (e.g., `src/components/SystemArchitect.tsx`).
2. **Explicit Action Markers**: Demarcate files with `[NEW]`, `[MODIFY]`, or `[DELETE]`.
3. **No Hidden Assumptions**: Inspect full imports and type signatures before writing code.
4. **Mandatory Verification**: Every generated prompt ends with explicit verification commands (`npx tsc --noEmit` and `npx vitest run`).

## 📐 3. Agentic Workflow Architecture Patterns

Artix supports 6 multi-agent architectural blueprints:

1. **Sequential Pattern**: Chain of agents passing refined outputs step-by-step.
2. **Parallel Fan-Out Pattern**: Distribute sub-tasks across concurrent agents and aggregate results.
3. **Orchestrator-Workers Pattern**: Central coordinator delegating to specialized worker agents.
4. **Router Pattern**: Directs inputs dynamically to specialized agent handlers.
5. **Evaluator-Optimizer Pattern**: Iterative generator-critic loop refining code until quality thresholds are met.
6. **Autonomous ReAct Pattern**: Reasoning + Action loop with tool calls.
